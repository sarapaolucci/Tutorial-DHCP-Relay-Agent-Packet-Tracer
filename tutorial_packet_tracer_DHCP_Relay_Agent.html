Obiettivi & Prerequisiti

Al termine di questo tutorial sarai in grado di:

  • Progettare e costruire una topologia a due reti in Packet Tracer con router, switch e PC.
  • Configurare un server DHCP che assegni indirizzi dinamici a tutti i client della rete.
  • Configurare le interfacce del router Cisco 2911 con indirizzamento IPv4.
  • Abilitare il DHCP relay (ip helper-address) per far raggiungere il DHCP server ai client di una subnet remota.
  • Verificare la corretta assegnazione degli indirizzi con strumenti come ipconfig e ping.

Prerequisiti

Prima di iniziare assicurati di conoscere i seguenti concetti:

  • Fondamenti del modello TCP/IP e indirizzi IPv4 con notazione CIDR.
  • Concetto di subnet, indirizzo di rete e broadcast.
  • Funzionamento del protocollo DHCP (DORA: Discover, Offer, Request, Acknowledge).
  • Interfaccia base di Cisco Packet Tracer (aggiunta dispositivi, cablaggio, CLI).
🔵 Nota sulle versioni
Il tutorial è stato sviluppato con Cisco Packet Tracer 8.x. Le interfacce grafiche possono variare leggermente nelle versioni precedenti, ma i comandi CLI rimangono identici.
01

Topologia di rete

La rete da realizzare è composta da due subnet separate e connesse da un unico router. Il server DHCP si trova sulla Rete 1 e serve dinamicamente i client di entrambe le reti grazie al relay configurato sull'interfaccia del router verso la Rete 2.

Piano di indirizzamento

DHCP Server FastEthernet0 192.168.10.2 255.255.255.0 192.168.10.1
Router 2911 GigabitEthernet0/0 192.168.10.1 255.255.255.0 —
Router 2911 GigabitEthernet0/1 192.168.20.1 255.255.255.0 —
PC-1, PC-2 FastEthernet0 Dinamico DHCP 255.255.255.0 192.168.10.1
PC-3, PC-4 FastEthernet0 Dinamico DHCP 255.255.255.0 192.168.20.1

⚠️ Attenzione agli IP esclusi
Gli indirizzi 192.168.10.1 (gateway router) e 192.168.10.2 (server DHCP) si trovano fuori dallo scope (.16 – .31), quindi non vengono assegnati dinamicamente. Non è necessario configurare esclusioni manuali.
02

Creazione del progetto in Packet Tracer

  1. Avvia Cisco Packet Tracer. Nella schermata iniziale clicca su File → New (o premi Ctrl+N) per creare un nuovo progetto vuoto.
  2. Vai su File → Save As e salva il file con un nome significativo, ad esempio rete_dhcp_relay.pkt, in una cartella dedicata al tuo lavoro.
  3. Verifica che la modalità attiva sia Realtime Mode (in basso a destra, l'icona con l'orologio verde). Non usare la Simulation Mode in questa fase.
03

Aggiunta dei dispositivi al workspace

Tutti i dispositivi si trovano nella barra in basso di Packet Tracer, suddivisi per categoria. Trascina ciascun dispositivo nell'area di lavoro posizionandolo in modo da rispettare la topologia mostrata nella sezione 01.

  2. Seleziona la categoria Switches e trascina due switch: posiziona Switch 1 a sinistra del router e Switch 2 a destra.

04

Cablaggio della rete

Per collegare i dispositivi usa il tipo di cavo corretto. In Packet Tracer seleziona il cavo dalla barra in basso (icona del fulmine/cavo), poi clicca sul primo dispositivo scegliendo la porta e poi sul secondo.

🔵 Cavo da usare
Usa sempre il Copper Straight-Through (cavo dritto, colore nero) per collegare dispositivi di tipo diverso (PC→Switch, Switch→Router, Server→Switch). Packet Tracer 8.x gestisce automaticamente anche i crossover, ma è buona abitudine usare il tipo corretto.

Configurazione del Server DHCP

Il server DHCP ha due compiti: possedere un IP statico e distribuire indirizzi dinamici ai client di entrambe le reti. In Packet Tracer il server si configura attraverso la GUI grafica.

5.1 – Assegnazione IP statico al server

  1. Fai doppio clic su DHCP-Server per aprire la finestra di configurazione.
  2. Vai nella scheda Desktop → IP Configuration.
  3. Seleziona Static e inserisci i seguenti valori:
CampoValore da inserire
IP Address [ 192.168.10.2 ]
Subnet Mask [ 255.255.255.0 ]
Default Gateway [ 192.168.10.1 ]

5.2 – Configurazione del servizio DHCP

  1. Nella stessa finestra del server, clicca sulla scheda Services → DHCP.
  2. Assicurati che il servizio sia On (pulsante verde in alto).
  3. Compila il primo pool di indirizzi con i seguenti valori e clicca su Add (o Save):
CampoValore da inserire
Pool NamePOOL_RETE1
Default Gateway [ 192.168.10.1 ]
DNS Server [ lasciare vuoto o 0.0.0.0 ]
Start IP Address [ 192.168.10.16 ]
Subnet Mask [ 255.255.255.0 ]
Maximum number of Users [ 16 ]
  1. Clicca Add per salvare il pool POOL_RETE1.
  2. Aggiungi un secondo pool per la Rete 2: clicca ancora su Add (o ripeti il procedimento). Compila il secondo pool:
CampoValore da inserire
Pool NamePOOL_RETE2
Default Gateway [ 192.168.20.1 ]
Start IP Address [ 192.168.20.16 ]
Subnet Mask [ 255.255.255.0 ]
Maximum number of Users [ 16 ]
✅ Perché due pool?
Il server DHCP identifica la subnet del client richiedente tramite l'IP sorgente della richiesta (o tramite il relay agent IP). Avendo due subnet distinte, occorrono due pool separati con i rispettivi gateway predefiniti. Il pool corretto viene selezionato automaticamente in base all'indirizzo di provenienza della richiesta.

06

Configurazione del Router 2911

Il router è il cuore della topologia: instrada il traffico tra le due reti e, grazie al comando ip helper-address, inoltrerà i broadcast DHCP provenienti dalla Rete 2 verso il server DHCP nella Rete 1.

⚠️ Interfacce spente per default
Le interfacce del router Cisco 2911 sono in stato administratively down per impostazione predefinita. Devi attivarle esplicitamente con il comando no shutdown.

6.1 – Accesso alla CLI del router

  1. Fai doppio clic sul router nel workspace.
  2. Clicca sulla scheda CLI. Se compare il messaggio "Would you like to enter the initial configuration dialog?", rispondi no e premi Invio.
  3. Premi Invio per accedere al prompt > (user EXEC mode).

6.2 – Configurazione interfaccia GigabitEthernet0/0 (Rete 1)

Router0 — Configurazione G0/0 (Rete 1) 
! Accede alla modalità privilegiata
Router> enable
Router# configure terminal
Router(config)# interface GigabitEthernet0/0
Router(config-if)# description Connessione a Rete 1 - 192.168.10.0/24
Router(config-if)# ip address [ INSERIRE: 192.168.10.1 ] [ INSERIRE: 255.255.255.0 ]
Router(config-if)# no shutdown
! Deve comparire: %LINK-5-CHANGED: Interface GigabitEthernet0/0, changed state to up
Router(config-if)# exit

07

Abilitazione del DHCP Relay (ip helper-address)

Normalmente un router blocca i pacchetti broadcast, tra cui le richieste DHCP Discover inviate dai client (destinazione 255.255.255.255). Il comando ip helper-address dice al router di intercettare quei broadcast sull'interfaccia specificata e di inoltrarli come unicast verso l'indirizzo del server DHCP.

📌 Dove si mette ip helper-address?
Il comando va inserito sull'interfaccia del router che è nella stessa rete dei client che non hanno ancora un IP. Nel nostro caso, i PC-3 e PC-4 sono nella Rete 2, quindi il relay va configurato sull'interfaccia GigabitEthernet0/1.
Router0 — DHCP Relay su G0/1 
Router# configure terminal
Router(config)# interface GigabitEthernet0/1
Router(config-if)# ip helper-address [ INSERIRE: 192.168.10.2 ]
! L'indirizzo è quello del DHCP Server nella Rete 1
Router(config-if)# end
Router# write memory
✅ Come funziona il relay
Quando un client sulla Rete 2 invia un DHCP Discover (broadcast), il router lo riceve su G0/1. Grazie al relay, il router crea un nuovo pacchetto unicast con: sorgente = IP dell'interfaccia G0/1 (192.168.20.1), destinazione = IP del server DHCP (192.168.10.2). Il server risponde al router, che poi smista la risposta al client corretto. Il server capisce a quale pool appartiene la richiesta grazie al campo relay agent IP address nel pacchetto.

Configurazione PC della Rete 1 (PC-1 e PC-2)

I PC della Rete 1 devono essere impostati su DHCP per ricevere automaticamente l'indirizzo IP dal server.

  1. Fai doppio clic su PC-1 per aprirne le proprietà.
  2. Vai nella scheda Desktop → IP Configuration.
  3. Seleziona l'opzione DHCP (il radio button in alto).
  4. Attendi qualche secondo: Packet Tracer simulerà il processo DORA. Vedrai comparire un IP nel range 192.168.10.16–192.168.10.31, la subnet mask 255.255.255.0 e il gateway 192.168.10.1.
  5. Ripeti gli stessi passi per PC-2.
🔵 Velocizzare la simulazione
Se l'IP non compare subito, clicca sul pulsante Fast Forward Time (doppia freccia avanti ▶▶) nella barra in basso. Questo accelera il tempo simulato e forza il completamento del handshake DHCP.

09

Configurazione PC della Rete 2 (PC-3 e PC-4)

La configurazione è identica a quella dei PC della Rete 1. La differenza è che questi PC riceveranno un indirizzo dalla subnet 192.168.20.0/24 grazie al relay configurato sul router.

  1. Fai doppio clic su PC-3.
  2. Vai in Desktop → IP Configuration e seleziona DHCP.
  3. Attendi l'assegnazione: il PC riceverà un IP nel range 192.168.20.16–192.168.20.31, subnet mask 255.255.255.0, gateway 192.168.20.1.
  4. Ripeti per PC-4.
📌 Cosa succede dietro le quinte
PC-3 invia un broadcast DHCP Discover. Lo Switch 2 lo inonda su tutte le porte. Il router lo riceve su G0/1 e, grazie a ip helper-address, lo incapsula in un pacchetto unicast verso 192.168.10.2. Il server DHCP riceve la richiesta, guarda il campo giaddr (192.168.20.1), seleziona il POOL_RETE2 e risponde con un indirizzo della Rete 2. Il router riceve la risposta e la inoltra a PC-3.

Test e verifica della rete

Questa fase è cruciale: non basta configurare, occorre verificare che tutto funzioni correttamente. Esegui tutti i test seguenti.

Test A – Verifica assegnazione IP con ipconfig

  1. Apri il PC-1 → Desktop → Command Prompt.
  2. Digita ipconfig e premi Invio.
  3. Verifica che compaia: IP 192.168.10.16 (o successivo), Mask 255.255.255.0, Gateway 192.168.10.1.
  4. Ripeti per PC-2, PC-3 e PC-4. I PC sulla Rete 2 devono avere IP nella subnet 192.168.20.x.

Test B – Ping al gateway

  1. Dal Command Prompt di PC-1, digita: ping 192.168.10.1
  2. Devono arrivare 4 risposte con Reply from 192.168.10.1.
  3. Dal Command Prompt di PC-3, digita: ping 192.168.20.1
  4. Stesso risultato atteso.

Test C – Ping cross-subnet (PC-1 → PC-3)

  1. Dal Command Prompt di PC-1, digita: ping 192.168.20.16 (o l'IP effettivo di PC-3).
  2. Devono arrivare 4 risposte: questo conferma che il router instrada correttamente il traffico tra le due subnet.

Riepilogo e concetti chiave

Hai completato con successo la configurazione di una rete con DHCP centralizzato e relay inter-subnet. Ecco i punti principali da ricordare:

  • Server DHCP con IP statico: il server stesso deve avere un indirizzo fisso e conoscere il proprio gateway per poter rispondere alle richieste smistate dal router.
  • Due pool DHCP separati: ogni subnet necessita di un pool dedicato con il proprio gateway predefinito. Il server associa la richiesta al pool corretto tramite il campo giaddr della richiesta.
  • Interfacce router: no shutdown: le interfacce Cisco sono spente di default; va sempre abilitata manualmente con no shutdown.
  • DHCP Relay (ip helper-address): il comando si inserisce sull'interfaccia del router affacciata sulla rete senza server DHCP locale. Converte il broadcast DHCP in unicast verso il server.
  • Verifica sistematica: usare ipconfig, ping e la Simulation Mode per confermare ogni singolo aspetto della configurazione.

Domande di verifica

  1. Perché il server DHCP non può avere un indirizzo assegnato dinamicamente dallo stesso pool che gestisce?
  2. Cosa accadrebbe se dimenticassimo di inserire ip helper-address sull'interfaccia G0/1 del router?
  3. Qual è il significato del campo giaddr (Gateway IP Address) nel pacchetto DHCP?
  4. Potresti aggiungere una terza rete alla topologia senza aggiungere un secondo server DHCP? Come?
  5. Cosa cambia nel funzionamento del relay se lo scope DHCP non copre la subnet della Rete 2?
🎯 Estensione opzionale
Prova ad aggiungere un DNS Server statico nella Rete 1 e a configurarne l'IP nei pool DHCP. Poi testa la risoluzione dei nomi aprendo il browser su uno dei PC e navigando verso un indirizzo simbolico.
